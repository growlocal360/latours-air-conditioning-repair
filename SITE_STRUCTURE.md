## Internal Linking Strategy

### Silo Rules:
1. **Homepage** links to all category silo pages
2. **Category pages** link back to homepage and to their child service pages
3. **Service pages** link back to their parent category page and sibling services
4. **Location pages** link to homepage and relevant service pages
5. **Neighborhood pages** link to homepage, are listed on homepage, and link to each other

### Cross-Linking:
- Each page contains NAP (Name, Address, Phone)
- Each page links to /schedule-service/
- Footer contains all major navigation links
- Breadcrumbs on all pages

---

## Schema Markup Strategy

1. **LocalBusiness** schema on every page
2. **Service** schema on service pages
3. **FAQPage** schema where applicable
4. **Review/AggregateRating** schema
5. **BreadcrumbList** schema
6. **Organization** schema

## Content Requirements Per Page

### Homepage (1500-2000 words)
- H1: Primary category + City
- H2s for each secondary category (with brief description)
- List of neighborhoods served (with links)
- Trust signals (years in business, reviews, certifications)
- FAQ section
- CTA buttons

### Category Pages (1000-1500 words)
- H1: Category + City
- H2s for each service in the category
- Link to all child service pages
- FAQ section
- Reviews/testimonials
- CTA buttons

### Service Pages (800-1200 words)
- H1: Service + City
- Detailed service description
- Benefits and features
- Process/what to expect
- Pricing information (if applicable)
- FAQ section
- Related services
- CTA buttons

### Location Pages (800-1200 words)
- H1: Primary Category + City/Town
- Local-specific content
- Distance from main location
- Services available
- Local landmarks/references
- CTA buttons

### Neighborhood Pages (600-800 words)
- H1: Primary Category + Neighborhood + Lake Charles
- Neighborhood-specific content
- Why residents trust Latour's
- Quick response times to that area
- Link back to main Lake Charles page
- CTA buttons
